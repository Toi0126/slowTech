# 仕様概要（MVP）

## 目的
親が録画や録音データをUpload、もしくは文字で文章を入力して、それを子供向けな言葉で文字起こし・要約し、こどもに感情を伝えるWebアプリを作成する。

本MVPでは、まず「入力（動画/音声/テキスト）のアップロード受付」と「データ保持」までを実装範囲とする。
文字起こし・要約の生成処理は将来対応とし、処理中/結果画面はダミー表示で代替する。

## 画面構成
- Upload画面
  - 録画・録音データをUploadする。
  - もしくはテキストを直接入力する。
  - Upload完了後、処理中画面へ遷移する。
- 処理中画面
  - 処理の進捗を表示する（MVPではダミー）。
  - 一定時間後に結果画面へ遷移する（MVPではダミー）。
- 結果画面
  - アップロードされた入力に対する結果を表示する（MVPではダミー）。
  - 結果をダウンロードできるリンクを提供する（MVPではダミー）。

## 入力仕様（MVP）
- 動画/音声ファイルのアップロードと、テキストの直接入力の両方に対応する。
- 動画/音声とテキストは「どちらか片方のみ」でも送信できる。

## 制約（MVP）
- 動画/音声の長さは「iPhoneから約2分の動画をアップロードできる程度」を目安とする。
  - 具体的な最大ファイルサイズ（MB）や許可形式（拡張子/Content-Type）は、実装着手時に確定する。

## データ保持（TTL）
- アップロードされた入力（動画/音声/テキスト）およびメタデータは1週間保持し、期限後は自動削除する。
  - 自動削除はベストエフォートとし、削除タイミングに多少の遅延が生じうる。

## 非スコープ（MVP）
- 文字起こし/要約の生成処理
- ログイン/アカウント登録
- 履歴一覧、共有リンク、通知

## 受け入れ条件（MVP）
- 動画/音声、またはテキストの送信が完了し、処理中画面（ダミー）へ遷移できる。
- 送信が成功した場合、結果画面（ダミー）へ遷移できる。
- 入力が制約を超過した場合、ユーザーにエラーを表示できる。
